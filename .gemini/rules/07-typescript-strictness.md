# TypeScript 엄격성 규칙 (Strictness)

- **`any` 타입 사용 금지**: 모든 변수, 함수 인자, 반환 값에는 명시적인 타입을 지정해야 합니다. `any` 타입을 사용하면 TypeScript의 핵심적인 장점인 타입 안정성이 저해되므로, 절대 사용해서는 안 됩니다. 불가피한 경우 `unknown`을 사용하고, 타입 가드를 통해 타입을 좁혀서 사용해야 합니다.

- **라이브러리 타입 검증**: 외부 라이브러리의 타입, 메서드, 속성을 사용하기 전에 반드시 해당 라이브러리의 타입 정의(d.ts) 파일을 확인하거나, 공식 문서를 통해 정확한 시그니처를 검증해야 합니다. 추측에 기반한 코드 작성을 금지하며, 라이브러리가 제공하는 타입을 100% 활용하여 타입 안정성을 보장해야 합니다.

- **타입 단언(Type Assertion) 최소화**: `as` 키워드를 사용한 타입 단언은 꼭 필요한 경우에만 최소한으로 사용해야 합니다. 타입 단언은 컴파일러의 타입 검사를 무력화시킬 수 있으므로, 제네릭, 타입 가드, `satisfies` 연산자 등을 통해 더 안전한 방법으로 타입을 추론하거나 좁힐 수 있는지 먼저 고려해야 합니다.

- **`@ts-ignore` / `@ts-expect-error` 금지**: 이 주석들은 타입 에러를 의도적으로 무시하는 행위이므로, 원칙적으로 사용을 금지합니다. 에러가 발생하면 근본적인 원인을 해결해야 합니다.
