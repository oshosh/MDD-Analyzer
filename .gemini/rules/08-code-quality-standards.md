# 좋은 코드를 위한 4가지 기준

제공해주신 자료를 바탕으로 좋은 프론트엔드 코드를 결정하는 4가지 핵심 기준을 요약해 드립니다.

### 1. 가독성 (Readability)
코드가 읽기 쉽고 의도를 파악하기 용이한 정도를 의미합니다.
*   **핵심:** 읽는 사람이 고려해야 할 맥락을 줄이고, 위에서 아래로 자연스럽게 읽히도록 구성합니다.
*   **전략:** 구현 상세 추상화, 복잡한 조건 및 매직 넘버에 이름 붙이기, 삼항 연산자 단순화 등.

### 2. 예측 가능성 (Predictability)
함수나 컴포넌트가 이름과 파라미터만 보고도 예상한 대로 동작하는 정도를 의미합니다.
*   **핵심:** 일관적인 규칙을 따르며, 숨은 로직 없이 명확한 반환 타입을 유지합니다.
*   **전략:** 이름 중복 방지, 반환 타입 통일, 숨겨진 로직 드러내기 등.

### 3. 응집도 (Cohesion)
수정되어야 할 코드가 항상 함께 수정되는지를 의미합니다.
*   **핵심:** 한 부분을 수정했을 때 의도치 않은 다른 곳의 오류를 방지하기 위해 관련 로직을 구조적으로 결합합니다.
*   **전략:** 관련 파일을 동일 디렉토리에 배치, 공통화 및 추상화를 통한 매직 넘버 제거 등.

### 4. 결합도 (Coupling)
코드를 수정했을 때 영향을 미치는 범위의 정도를 의미합니다.
*   **핵심:** 수정 시 영향 범위를 최소화하고 예측 가능하게 만들어 변경에 유연하게 대응합니다.
*   **전략:** 단일 책임 원칙 준수, 과도한 공통화 대신 적절한 중복 허용, Props Drilling 제거 등.

**결론:** 이 4가지 지표는 서로 상충(Trade-off)할 수 있으므로, 개발자는 상황에 따라 가독성과 응집도, 혹은 결합도 사이의 우선순위를 판단하여 **변경하기 쉬운 코드**를 작성해야 합니다.
