# UI 구현 규칙 (이미지 1:1)

- **Material 3 준수**: `src/app/globals.css`에 정의된 컬러 토큰(`--md-sys-color-*`)을 반드시 사용하며, Tailwind v4 `@theme` 블록과 통합하여 일관된 테마를 유지한다.
- **디자인 감성**: Glassmorphism, 정교한 그림자, 명확한 타이포그래피를 통해 "현대적 핀테크 대시보드" 미학을 유지한다.
- **반응형 및 모드**: 다크/라이트 모드를 기본 지원하며, 모바일 우선 대응(Mobile-First)을 원칙으로 한다.
- **이미지 1:1 구현**: 사용자 제공 엑셀/시트 이미지 구조와 결과를 100% 일치시키며, 설명용 자막/문구는 구현 대상에서 제외한다.
- 이미지에 등장한 모든 표는 **동일한 행/열/순서/구조**로 구현한다.
- 차트는 이미지에 있는 것과 동일한 개수/축/단위를 따른다.

# 모바일 레이아웃 및 시각적 검증 (Playwright MCP)

- **검증 의무**: 모바일 레이아웃 수정 시 반드시 Playwright MCP를 사용하여 실제 렌더링을 확인해야 한다.
- **타겟 뷰포트**: 
  - iPhone 13 (390x844)
  - iPhone XR (414x896) - 가장 널리 쓰이는 표준 좁은 화면 기준
- **오버플로우 탐색**: 횡 스크롤 흔들림 발생 시 브라우저 내에서 `clientWidth > viewportWidth` 요소를 탐색하여 빨간색 테두리(`border: 2px solid red`)를 부여하는 시각적 디버깅 기법을 적극 활용한다.
- **독립 스크롤 보장**: 전체 페이지가 수평으로 흔들리지 않아야 하며, 테이블과 같은 거대 요소는 반드시 개별 카드 내에서 독립적으로 횡 스크롤 되어야 한다.

# 반드시 포함할 UI 산출물

- 상단 요약 표(이미지의 표 구조 그대로)
- RAW 테이블(엑셀 RAW 시트와 동일 컬럼)
- 일별 MDD 차트 (USD 기준 / KRW 기준)
- 기간 누적 수익률(USD/KRW)
- 구간별 회복율 표(0, -5, -10, ... 이미지 동일 구간)

# 스키마 잠금 규칙 (포맷)

- RAW 컬럼명, 순서, 타입은 `docs/RAW_SCHEMA.md`와 반드시 동일해야 한다.
- UI 표 컬럼, 순서, 구간은 `docs/UI_TABLES.md`와 반드시 동일해야 한다.
- 퍼센트/소수점 자리수/천단위 구분/날짜 포맷은 엑셀과 동일하게 맞춘다.
- 빈 값은 빈 값으로 둔다. (임의로 0 채우기 금지).
- 변경이 필요하면 반드시 문서를 먼저 수정하고, 문서 없이 구조 변경을 금지한다.

# App Router 구조 (RSC 고정)

- **기본 원칙**: 모든 페이지는 서버 컴포넌트(RSC)를 기본으로 하며, 인터랙션이 필요한 말단 컴포넌트만 `"use client";`를 명시한다.
- **에러 및 예외 처리**:
  - `error.tsx`: 예상치 못한 런타임 에러 발생 시 사용자에게 재시도 버튼과 에러 메시지를 제공하는 **Error Boundary** 역할을 수행한다.
  - `not-found.tsx`: 존재하지 않는 자산(Ticker) 검색이나 잘못된 경로 접근 시 **404 페이지**를 렌더링한다. (서버/클라이언트 모두에서 `notFound()` 함수를 호출하여 트리거한다.)
  - **상태 코드**: API Route 에러는 400, 404, 502, 500 상태 코드를 명확히 구분하여 응답하며, UI는 이에 맞춰 적절한 피드백을 주어야 한다.
- **데이터 프리페치**: `page.tsx` 패턴은 `HydrationBoundary`와 `dehydrate` 패턴을 사용하여 서버에서 쿼리를 prefetch한다.
